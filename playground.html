👨‍🎓 Complete Beginner Tutorial

New to APIs? Follow this step-by-step guide to understand how EzyAPI makes everything simple!

📖 Step-by-Step Tutorial:

🏗️ Step 1: Create Your HTML Structure

First, create your HTML with special placeholders where you want data to appear:

<!-- inside body -->
<div id="profileCard">
  <img id="profilePic" src="{picture}" alt="Profile" />
  <h2 id="profileName">{name}</h2>
  <p id="profileEmail">{email}</p>
  <p id="profileCreated">Joined: {created_at}</p>
  <p id="given_name">{given_name}</p>
</div>

Notice: The curly braces {} are placeholders that will be automatically replaced with real data!

📚 Step 2: Include Required Scripts

Add these scripts to your HTML (Axios is optional but recommended):

<!-- Optional: Axios for better API handling -->
<script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>

<!-- Required: EzyAPI library -->
<script src="ezyapi.js"></script>
🚀 Step 3: Write Your JavaScript

Now the magic happens with just a few lines of code:

<script>
  // Method 1: One line does everything! (Recommended for beginners)
  autoBind("../../api/user-profile.php");
  
  // Method 2: Step by step (for learning/debugging)
  var apiCall = callapi("../../api/user-profile.php");
  console.log(apidata(apiCall)); // See the data in console
  
  // Method 3: Advanced usage with specific target
  autoBind("../../api/user-profile.php", document.getElementById('profileCard'));
</script>
🎯 What Happens:
  1. API Call: EzyAPI fetches data from your API endpoint
  2. Data Processing: Response is automatically processed and extracted
  3. DOM Binding: Placeholders like {name} are replaced with actual values
  4. Error Handling: Any errors are caught and handled gracefully
💡 Beginner Tips:
  • Placeholder Names: Must match your API response keys exactly
  • Nested Data: Use dots like {user.address.city} for nested objects
  • Missing Data: If API doesn't have a field, placeholder becomes empty (no errors!)
  • Console Debug: Use console.log(apidata(callapi('url'))) to see your API response
  • Local Testing: Make sure your API supports CORS for browser testing

🔥 Available Placeholder Patterns:

Simple Properties:
{name} - Direct property
{email} - User email
{id} - User ID
Nested Objects:
{user.name} - Nested property
{address.city} - Address city
{company.name} - Company name
Deep Nesting:
{profile.settings.theme}
{data.results.0.name}
{config.api.endpoints.user}
Optional Prefix:
{v.name} - Same as {name}
{v.user.email} - Optional v. prefix
{v.data.count} - For clarity

💡 Pro Tips:

  • Arrays: Use {items.0.name} for first item, {items.length} for count
  • Missing Data: Missing properties show as empty (no errors!)
  • Objects: Complex objects auto-convert to JSON strings
  • Multiple APIs: Call autoBind() multiple times for different sections
  • Custom Elements: Target specific divs: autoBind('/api', document.getElementById('myDiv'))
